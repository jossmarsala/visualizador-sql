📍 Subsistema Geográfico

Jerarquía de ubicación: una provincia tiene muchas localidades; una localidad ubica a los usuarios de SignAI.

👤 Usuarios & Planes de Suscripción

El corazón del modelo de negocio: usuarios registrados, planes disponibles y sus suscripciones activas. Un usuario puede tener múltiples suscripciones a lo largo del tiempo.

🤖 IA & Traducción de Señas

El motor central de SignAI: versiones del modelo de IA, plataformas soportadas, catálogo de señas y las sesiones de traducción que las detectan.

💳 Sistema de Facturación

Cada suscripción genera una factura. La factura tiene líneas de detalle y se puede pagar combinando distintas formas y medios de pago.

⭐ Feedback & Logs de Error

Calidad y trazabilidad: el usuario puntúa cada sesión (1:1) y el motor registra errores técnicos (1:N) para auditoría del equipo de desarrollo.

🌟 Módulo de Mejoras — Colecciones & Auditoría

Funcionalidades adicionales: colecciones personales de señas organizadas por el usuario, y registro de auditoría automático de cambios de precios en planes.

🔀 Tablas Puente (Muchos a Muchos)

Cuando dos entidades se relacionan en ambas direcciones se usa una tabla puente. SESION_SENA además guarda la confianza de reconocimiento de cada seña detectada.

🔗 Cadenas de Dependencia

Una tabla padre contiene datos que la tabla hija referencia mediante una clave foránea (FK). No podés borrar el padre sin antes atender a sus hijos.

PADRE tabla raíz, tiene PK · → depende de / referencia a · HIJO tiene FK al padre · PUENTE une dos tablas

🔀 Puentes Many-to-Many

Una sesión puede reconocer muchas señas, y una seña puede aparecer en muchas sesiones. Una colección personal puede contener muchas señas, y una seña puede estar en muchas colecciones.

🔒 Triggers — guardias automáticos

Un trigger es una regla que se ejecuta sola cuando ocurre un evento (insertar, borrar, actualizar). No necesitás llamarlo: la base de datos lo activa automáticamente como un guardia de seguridad.

⚡ Stored Procedures — acciones prearmadas

Un procedure es como una función guardada en la base de datos. Lo llamás pasando parámetros y él ejecuta una operación compleja de una sola vez, sin repetir SQL.

👁️ Views — ventanas de datos

Una view es una consulta SQL guardada con nombre propio. La usás como si fuera una tabla, pero combina datos de varias tablas en tiempo real. Perfecta para reportes sin duplicar datos.

🔒 Impacto de los Triggers

Los triggers actúan como barreras de seguridad invisibles. Se disparan solos cuando alguien intenta una operación prohibida.

INSERT en SESION_SENA
→
trg_validar_confianza_ia
→
❌ bloquea si confianza < 0 o > 100
DELETE en FACTURA
→
trg_evitar_borrado_factura_pagada
→
❌ bloquea si tiene pagos registrados
INSERT en SUSCRIPCION
→
trg_fechas_suscripcion_coherentes
→
❌ bloquea si fecha_fin ≤ fecha_inicio
UPDATE en PLAN_SUSCRIPCION
→
trg_prevenir_precio_negativo
→
❌ bloquea si precio_mensual < 0

❌ bloquea si tiene historial de facturas
